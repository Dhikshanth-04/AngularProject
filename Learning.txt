Why angular was created : 
    * Angular was created to manage the large dynamic web projects.
    * it introduces the structured, components-based framework with features like dependency injection, routing, reusable components, testing support, making applications easier to develop, maintain and scale

Difference between AngularJS and Angular :
    * AngularJS was the original framework based on javascript and based largely on controllers and digest cycles.
    * Angular 2 was based on components based architecture, wirtten using typescript.
    * Better performances, improved tooling and a design intended for large, scalable applications. 

Semantic Versioning : 
    * The Semantic Versioning is the process of naming the version, with the meaning full context, initial version: 1.0, small feature added, it would be 1.1, if the whole new design is produced, this is version 2.0. This naming process is known as semantic versioning. (major. minor. patch)

Evolution of ANGULAR : 

Angular 2 features:
    * component based architecture, which provides the more smaller broken modules instead of holding up the large blocks
    * It chooses the typescript, because of type safe, better performance, lrge team development.
    * more efficient change detection, than AngularJS

Angular 3?

Angular 9:

Angular 10 - 13: Improved the speed, stability, foundation of IVY compiler. 

Angular 14: 
    * Introduction of standalone components, before Angular14 every component belongs to the module, and also had to be Registered it inside @NgModule({})
    * It provides the simpler architecture, easier learning, better code organization. 
    * Reduces the dependency of modules, for simpler angular projects. Besides, reduces the boiler plate code and improved developer experience.

Angular 15: Improvised the performance of standalone components

Angular 16: 
    * Introduction of siganls, it mainly focuses on the changes happening around.
    * Earlier angular, mainly relied on change detection mechanism to determine what needed to be updated.
    * siganls as a new reactive primitive.

Angular 17: The templates looks cleaner and seems like a programming.

Angular 18 - 20: Server side improvement, signal improvement, performance.

Component : Its the small block of reusable code, works independently, and the part of the UI, like navrbar, search bar, footer, etc...

Modules : Its a logical container, that groups or organizes related anugalr building block together. For Example : Authentication module : login component, Register component, Forgot password component.

Bootstraping : Angular creates the root component and begins rendering UI.

Decorators : It gives the meta data, data about the data. 

NodeJs : Its a javascript runtime environment, that allows JS to run outside the browser.

Why angular needs NodeJs : angular application runs on browser, but angular development kit runs on nodejs. Angular uses NodeJs as the development environment.

Dependency tree : The series of dependencies linked to one to another. 

NgModule : Its the decorator object, which registers about the meta data. In this file, the components, imports, Declarations, providers and also specifies the bootstrap component will be mentioned, when the application starts.

Components :
    * A component is the reusable UI unit, it contains; HTML template that defines structure, CSS stylesheet the defiens the apperance and a typescript class contains the business logic, at last @component decorator that provides metadate telling angular how these pieces are connected.  
    * Each component has one responsibility, this is called as Single Responsibility Principle.

Angular CLI : 
    * Command Line Interface. Instead of clicking the buttons, we communicate through commands with software.
    * Its the official command line interface provided by the angular team, it automates the creation, development, testing, building and maintenance of angular applications through the commands. 
    * It does not runs the app on browser, it prepares and serves the application during the development. 

Commands of ANGULAR CLI : 
    * Creating new angular project : ng new `project name`. 
    here; ng - angular cli, new - new project, `project name`
    * Angular CLI starts, creates the project blueprint and the angular project will be created. 

NgModule : Angular decorator. @NgModule : it represents, this class is component 

Compilation process :
    ng serve -> angular CLI starts -> reads angular.json -> reads tsconfig.json -> reads main.ts -> angular compiler starts -> read all .ts files -> understands components -> processes angular templates -> compile ts to js -> bundle html, css -> creates optimised file -> development server starts -> browser downloads html, css, js -> browser executes js -> application appears.

Project architecture : 
    1) package.json : 
        * Its a the configuration file that contains the projct information, dependencies, scripts, version info and other meta data required to manage the project.
        * name - name of the project
        * version - version of the project, semantic versioning (major. minor. patch)
        * scripts - shortcuts of the commands.
        * private : true : It prevents accidental publishing
        * dependencies - the list of packages required for the application to work.
        * devDependencies - the tools thats required by the developers - typescript, angular, etc...
        * @angular-core : contains all about angular, heart of angular. It contains : components, decorators, Dependency injection and change detection 
        * @anugalr-common : commonly used features like, *ngIf, *ngFor 
        * @angular-router : Used for navigation.
        * rxjs : Library file used for asynchronous data and stream of values.
        * tslib : contains helper functions for typescript compilation.
        * Zone.js : it helps angular to know, when the asynchronous operation complete and allowing angular to update the UI at the right time. 
    
    2) node modules : 
        * The folder where the installed packages will be stored around.
        * Its a huge folder, because the dependencies are linked to one another.

    3) package-lock.json : It stores the exact version of installed packages and their dependency tree, ensuring consistent installations for different environments. 

    workflow : Developer -> npm install -> reads package.json -> reads package-lock.json(if present) -> npm registry -> download exact version -> stroes in node modules -> app ready.  

    4) angular.json :
        * Its a wrokspace configuration file. It tells the angular CLI how to build, serve, test, manage the application. Its like project settings.
        * ng serve -> Angular CLI -> angular.json -> finds project settings -> starts build
        * $schema : it helps the vs code to understand the angular.json
        * angular CLI heavily depends on this file. It helps to add global dependencies.
        * "build" :{...} : it compiles the angular code. ng serve -> angular CLI -> angular.json -> builder -> compiles angular -> browser
        * output path : Once the files are compiled, stroed in the dist folder. 
        * main.ts : starting point of the execution. main.ts -> tsconfig.app.json -> typescript compiler -> javascript.

    5) Complete execution : 
        ng serve -> package.json -> uses installed angular packages ->reads angular.json -> looks for  builder -> finds @angular-dev-kit/build-angular -> reads tsconfig.app.json -> compiles main.ts -> processes angular components -> ts to js -> starts development server -> browser opens in 4200 -> browser downloads html, css, js -> application appears. 

    6) tsconfig.json : 
        * its the configuration file for the typescript compiler, to specify how the ts code should be compiled to js. 
        * compileOn : false : If the file saved, the typescript automatically compiles or not.
        * compilerOptions : target(JS version), module(allows import and export JS files), strict(catches error during execution), noImplicitAny(data types in parameters of function), experimentalDecorators(it allows the decorators)
        * It verifies the base configuration.
    
    7) tsconfig.app.json : Rules for the application
    
    8) tsconfig.spec.json : Rules for testing

    9) main.ts : 
        * Its the entry point of the execution
        * import { platformBrowserDynamic } from '@angular/platform-browser-dynamic';
        here; import - bring something from another module, platformBrowserDynamic - function provided by angular found in package.json,  platformBrowserDynamic() - function prepares the angular to run inside a browser
        * import { AppModule } from "./app/app.module.ts";   
        here; AppModule - root module of angular(not component, because angular app starts with module)
        * platformBrowserDynamic().bootstrapmodule(AppModule) - Heart of angular
        here, Bootstraping : Process of starting or initializing.
        * AppModule - the file contains everything like, components, pipes, directives, alll. AppModule -> find app components -> create app components -> render html -> browser
        * Execution flow : ng serve -> package.json -> angular.json -> tsconfig.json -> compile ts to js -> main.ts -> platformBrowserDynamic() -> prepares the browser -> AppModule -> App components -> HTML -> browser displays UI.

    10) app.module.ts :
        * Root module, it organizes an angular application's components, imported modules, providers, specifies bootstrap components.
        * NgModule : Because its used to represent, that this class is component
        * BrowserModule : Provides essential services and directives needed for an angular application running in a browser. An object, everything inside this would describe the application.
        * NgModule is the decorator, that contains the registration meta data. It tells the angular which components, directives, providers, imports, bootstrap are belongs tot he module. 
        * Declaration :[] -> what component, belongs to this module

    11) app.component.ts : 
        * Its the business logics, variables and wrokspace, 
        * @component : it contains - selector, templateUrls, styleUrls and the type script class.
        * selector : Its denotes a html tag that represents the component
        * templateUrls : it relates the html file belongs to the component
        * styleUrls : it relates the css file belongs to the component
        * Its a decorator object, contains only information, business logics goes inside the class.
    
    12) app.component.html : It contains the html tags and display the UI

    13) app.component.css : It contains the styles regarding to the component

    14) index.html : Browser always starts executes the index.html, it contains something <app-root></app-root>, its the selector in component of app.component.ts, which has current html, css and typescript business logic.

Data binding is the mechanism by which angular synchronizes data between the component and template. it allows data to flow from template to component, component to template or both, there are two different types;
    1. One way binding
    2. Two way binding

One way binding has 3 different methods;
    * Interpolation => component - template => {{}} - Reads and displays value
    * Property binding => component - template => [] => bind data from component to html-element Property
    * Event binding => template - component => ()

Two way binding has only one method : ngModel => [(ngModel)]="name". The data would be passed from component - template and template - component. To use this model, we need to import the library of `FormsModule` 

Why data binding : To eliminate the manual dom manipulation, reduces repetitive code, improve maintainability and automatically synchronise the UI with the component data.

Interpolation and this : Angular templates are evaluated in the context of current component instance. this is unnecessary and not allowed. 

common properties in property binding : [disabled], [checked], [hidden], [colspan], [rowspan], [max], [min]

Two way binding : 
    * Two way binding synchronises data automatically between the components and view. Any change in view updates components and changes in components updates view. 

Directives:
    * Changes the behaviour or apperance of this html element automatically.
    * Directives are the class in angular, which provies additional behaviour, structure or apperance to the DOM elements.
    * Directives are angular code that works on HTML.
    * NgIf, NgFor, NgSwitch - contains in @angular/common

Component directive :
    * A component is the special type of directive that has its own template.
    * Component and directives act together to enhance the element
    * Directives enhaces the existing element
    * Component provides the whole template.

structural directives : 
    * These are the directives that add, remove, or manipulate the HTML elements, thereby changing the DOM tree.
    * The (*) is the expansion of the <ng-template></ng-template>
    * (*ngIf="conditon") : Only if the value, states true, the element will be created, else not created.
    * It does not makes the visiblity as none, the node itself not created.

ng-template : Its an angular template container, that is not rendered immediately. Angular creates and renders only if its necessary.

*ngFor="" : 
    * its a structural directive used to repeat the html element for every item in collection such as an array.

    <h2 *ngFor="let i of arr">
    {{i}}</h2>

*ngSwitch="" :
    * Its a structural directive, that selects one template from multiple possible templates based on the expression. 

Attribute directives:
    * This attributes changes the apperance or behaviour of the html element, without adding or removing it from the tree.

ngClass : Its an attribute directive, in which the css classes will be added or removes based on angular expressions. 

ngStyle : Its an attribute directive, that dynamically applies one or more inline css to the existing HTML based on angular expression.

Services : 
    * A servies is a typescript class that contains the business logic, data access, API calls, and shared functionality that can be used by mulitple components created by using @Injectable. 

    * Component coordinates the UI and the services does the work.

    * Services are purely the typescript functions, no HTML no CSS.

    * Its used to separate the business logics from UI, Reduces the code duplication, Improve code maintainability and reusablity

Dependency Injection : 
    * An object or the class that another class needs in order to perform its work.     
    * Dependency Injection is a design pattern where an object receives its required dependencies from an external provider instead of creating them itself. 

Constructor injection : Its the process of receiving dependencies through the Constructors of the class. Angular injects the required services into components when creating the component. 

service(business logic) -> Dependency(Object that othr class needs) -> Dependency injection(Angular provides the object) -> Constructor Injection(Providing the object through the Constructor).

@Injectable({
    providedIn : 'root';
})
here; 
    * @Injectable is the decorator, says that this class is the service
    * providedIn : 'root' -> Create a service for the entire application
    * main.ts is the root of the app. 

Import Vs Inject
    * Import is just to tell class exist, does not creates or provides object. Used to use the class.
    * Inject is the angular command, provides the instance of the class. Used to use the service object.

Pipe : 
    * Its an angular feature used to transform or format data in the template before displaying it to the user without modifying the original data.  
    * It has different types like; uppercase, lowercase, percent, currency, json, key-value pair, async too.

Custom pipes : These are the user defined pipes created to transform the data, according to the application requirements. These pipes are used, when the built-in types are not satisfied. 

Angular forms : 
    * Instead of writing logics in html, everything will be wirtten in TS. Business logic -> TS -> HTML.
    * Angular creates one object - FormGroup, inside - FormControls.
    * Each fields are one FormControl....
    * A reactive forms are model-driven approach in angular where the entire form and its validations are created and managed inside the typescript component instead of html template.

FormControls : These are objects that manages the textbox. It keeps the track of state. State is whats happening in the input box right now. 

FormGroup : It groups the related controls. 

FormBuilder : Its like a helper, builds FormGroups. 

?. : This notation is called as optional chaining, If the control exists, return the value else returns 'undefined'.

Touched : Initially false, when the user focuses on the input field and leaves. It becomes true. 

Dirty : The value of the form control has been changed from its original value. 

Validation : The process of checking whether the user statisfies the required rules before processing or storing the data. 

Cross field validation : One form control is verified with other form control.

Form builder : Its a service that simplifies the creation of formgroup, form control and form array. It internally creates the same reactive objects, but with much cleaner syntax. 

setValue(): It is used to set values for all controls  in a form.
patchValue(): It is used set the values for specific fields

@Input() : Its a decorator, used to receive data from a parent component to child component.

@Output : ITs the decorator,  which is used to sedn data or event from child to parent component.

EventEmitter : Its an angular class used to emit custom data or event to the parent class from the child class.

emit() : It sends the data through the EventEmitter to the parent component

$event : Its the special angular variable that contains the value emitted by the child component. 

Routing : Its the process of navigating thorugh the components, within the single page application, without reloading the entire webpage.

Angular - SPA : Angular is the single page application, where only one html page will be loaded initially, and different components are loaded dynamically, without loading the whole web page.

Programmatic navigation : importing the router, defining the constructor, navigate. Instead of clicking the links, once the login is done, the home would be navigated. 

<router-outlet></router-outlet> : Its the reserved space.
