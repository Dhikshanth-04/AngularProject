Why angular was created : 
    * Angular was created to manage the large dynamic web projects.
    * it introduces the structured, components-based framework with features like dependency injection, routing, reusable components, testing support, making applications easier to develop, maintain and scale

Difference between AngularJS and Angular :
    * AngularJS was the original framework based on javascript and based largely on controllers and digest cycles.
    * Angular 2 was based on components based architecture, wirtten using typescript.
    * Better performances, improved tooling and a design intended for large, scalable applications. 

Semantic Versioning : 
    * The Semantic Versioning is the process of naming the version, with the meaning full context, initial version: 1.0, small feature added, it would be 1.1, if the whole new design is produced, this is version 2.0. This naming process is known as semantic versioning. (major. minor. patch)

Evolution of ANGULAR : 

Angular 2 features:
    * component based architecture, which provides the more smaller broken modules instead of holding up the large blocks
    * It chooses the typescript, because of type safe, better performance, lrge team development.
    * more efficient change detection, than AngularJS
    * Built with the innovation of modern web and mobile in mind.

Angular 3?
    * Angular 3 was intentionally skipped to synchronise the versions name, where some of angular packages like router were already reached version 3.

Angular 4: Better performance, improved animations, smaller bundle sizes, fast rendering.

Angular 5: Developer focused - fast to build, easier to ship, better compiler support.

Angular 6: Angular CLI automated, the generation of components through commands. Made libraries easier to share - Angular package format.

Angular 7: performance, virtual scrolling, Drag and drop support through Component Dev Kit.

Angular 8: Lazy loading improvements, loads the page only when the user needs it.

Angular 9: 
    * The introduction of IVY compiler, Angular's modern compiler and rendering engine, which improved the compilation speend, reduced the bundle size, better for debugging and made applications faster and more efficient.

Angular 10 - 13: Improved the speed, stability, foundation of IVY compiler. 

Angular 14: 
    * Introduction of standalone components, before Angular14 every component belongs to the module, and also had to be Registered it inside @NgModule({})
    * It provides the simpler architecture, easier learning, better code organization. 
    * Reduces the dependency of modules, for simpler angular projects. Besides, reduces the boiler plate code and improved developer experience.

Angular 15: Improvised the performance of standalone components

Angular 16: 
    * Introduction of siganls, it mainly focuses on the changes happening around.
    * Earlier angular, mainly relied on change detection mechanism to determine what needed to be updated.
    * siganls as a new reactive primitive.

Angular 17: The templates looks cleaner and seems like a programming.

Angular 18 - 20: Server side improvement, signal improvement, performance.

Component : Its the small block of reusable code, works independently, and the part of the UI, like navrbar, search bar, footer, etc...

Modules : Its a logical container, that groups or organizes related anugalr building block together. For Example : Authentication module : login component, Register component, Forgot password component.

Bootstraping : Angular creates the root component and begins rendering UI.

Decorators : It gives the meta data, data about the data. 

NodeJs : Its a javascript runtime environment, that allows JS to run outside the browser.

Why angular needs NodeJs : angular application runs on browser, but angular development kit runs on nodejs. Angular uses NodeJs as the development environment.

Dependency tree : The series of dependencies linked to one to another. 

NgModule : Its the decorator object, which registers about the meta data. In this file, the components, imports, Declarations, providers and also specifies the bootstrap component will be mentioned, when the application starts.

Components : These are smaller reusable parts of the User Interface(UI) by combining a typescript class, html template and css styles. Each component has one responsibility, this is called as Single Responsibility Principle.

Angular CLI : 
    * Command Line Interface. Instead of clicking the buttons, we communicate through commands with software.
    * Its the official command line interface provided by the angular team, it automates the creation, development, testing, building and maintenance of angular applications through the commands. 
    * It does not runs the app on browser, it prepares and serves the application during the development. 

Commands of ANGULAR CLI : 
    * Creating new angular project : ng new `project name`. 
    here; ng - angular cli, new - new project, `project name`
    * Angular CLI starts, creates the project blueprint and the angular project will be created. 

NgModule : Angular decorator. @NgModule : it represents, this class is component 

Compilation process :
    ng serve -> angular CLI starts -> reads angular.json -> reads tsconfig.json -> reads main.ts -> angular compiler starts -> read all .ts files -> understands components -> processes angular templates -> compile ts to js -> bundle html, css -> creates optimised file -> development server starts -> browser downloads html, css, js -> browser executes js -> application appears.

Project architecture : 
    1) package.json : 
        * Its a the configuration file that contains the projct information, dependencies, scripts, version info and other meta data required to manage the project.
        * name - name of the project
        * version - version of the project, semantic versioning (major. minor. patch)
        * scripts - shortcuts of the commands.
        * private : true : It prevents accidental publishing
        * dependencies - the list of packages required for the application to work.
        * devDependencies - the tools thats required by the developers - typescript, angular, etc...
        * @angular-core : contains all about angular, heart of angular. It contains : components, decorators, Dependency injection and change detection 
        * @anugalr-common : commonly used features like, *ngIf, *ngFor 
        * @angular-router : Used for navigation.
        * rxjs : Library file used for asynchronous data and stream of values.
        * tslib : contains helper functions for typescript compilation.
        * Zone.js : it helps angular to know, when the asynchronous operation complete and allowing angular to update the UI at the right time. 
    
    2) node modules : 
        * The folder where the installed packages will be stored around.
        * Its a huge folder, because the dependencies are linked to one another.

    3) package-lock.json : It stores the exact version of installed packages and their dependency tree, ensuring consistent installations for different environments. 

    workflow : Developer -> npm install -> reads package.json -> reads package-lock.json(if present) -> npm registry -> download exact version -> stroes in node modules -> app ready.  

    4) angular.json :
        * Its a wrokspace configuration file. It tells the angular CLI how to build, serve, test, manage the application. Its like project settings.
        * ng serve -> Angular CLI -> angular.json -> finds project settings -> starts build
        * $schema : it helps the vs code to understand the angular.json
        * angular CLI heavily depends on this file. It helps to add global dependencies.
        * "build" :{...} : it compiles the angular code. ng serve -> angular CLI -> angular.json -> builder -> compiles angular -> browser
        * output path : Once the files are compiled, stroed in the dist folder. 
        * main.ts : starting point of the execution. main.ts -> tsconfig.app.json -> typescript compiler -> javascript.

    5) Complete execution : 
        ng serve -> package.json -> uses installed angular packages ->reads angular.json -> looks for  builder -> finds @angular-dev-kit/build-angular -> reads tsconfig.app.json -> compiles main.ts -> processes angular components -> ts to js -> starts development server -> browser opens in 4200 -> browser downloads html, css, js -> application appears. 

    6) tsconfig.json : 
        * its the configuration file for the typescript compiler, to specify how the ts code should be compiled to js. 
        * compileOn : false : If the file saved, the typescript automatically compiles or not.
        * compilerOptions : target(JS version), module(allows import and export JS files), strict(catches error during execution), noImplicitAny(data types in parameters of function), experimentalDecorators(it allows the decorators)
        * It verifies the base configuration.
    
    7) tsconfig.app.json : Rules for the application
    
    8) tsconfig.spec.json : Rules for testing

    9) main.ts : 
        * Its the entry point of the execution
        * import { platformBrowserDynamic } from '@angular/platform-browser-dynamic';
        here; import - bring something from another module, platformBrowserDynamic - function provided by angular found in package.json,  platformBrowserDynamic() - function prepares the angular to run inside a browser
        * import { AppModule } from "./app/app.module.ts";   
        here; AppModule - root module of angular(not component, because angular app starts with module)
        * platformBrowserDynamic().bootstrapmodule(AppModule) - Heart of angular
        here, Bootstraping : Process of starting or initializing.
        * AppModule - the file contains everything like, components, pipes, directives, alll. AppModule -> find app components -> create app components -> render html -> browser
        * Execution flow : ng serve -> package.json -> angular.json -> tsconfig.json -> compile ts to js -> main.ts -> platformBrowserDynamic() -> prepares the browser -> AppModule -> App components -> HTML -> browser displays UI.

    10) app.module.ts :
        * Root module, it organizes an angular application's components, imported modules, providers, specifies bootstrap components.
        * NgModule : Because its used to represent, that this class is component
        * BrowserModule : Provides essential services and directives needed for an angular application running in a browser. An object, everything inside this would describe the application.
        * NgModule is the decorator, that contains the registration meta data. It tells the angular which components, directives, providers, imports, bootstrap are belongs tot he module. 
        * Declaration :[] -> what component, belongs to this module

Data binding is the mechanism by which angular synchronizes data between the component and view. it allows data to flow from view to component, component to view or both, there are two different types;
    1. One way binding
    2. Two way binding

One way binding has 3 different methods;
    * Interpolation => component - view => {{}}
    * Property binding => component - view => []
    * Event binding => view - component => ()

Two way binding has only one method : ngModel => [(ngModel)]="name". The data would be passed from component - view and view - component. To use this model, we need to import the library of `FormsModule`
